### Terminal
Atualmente estou usando o **Terminal padrão do Mac**, mas se você já estiver enjoado, recomendo o [Warp](https://warp.dev/), ele tem IA integrado então caso esqueça um comando fica fácil de relembrar. Usei ele por um tempo e é muito bom, com um design minimalista mt bonito. Outro indicação é o [WezTerm](https://wezterm.org/) esse eu gosto de usar no Windows.

### Navegador
Antes eu estava utilizando o [Arc](https://arc.net/), apesar de ser um navegador bem legal e completo, ver que agora eles estão focando em um novo e deixando o Arc meio que esquecido, fez eu voltar para o bom e velho [Google Chrome](https://www.google.com/intl/pt-BR/chrome/).

### Stack

[Vite](https://vite.dev/) – Costumo utilizá-lo para páginas SPA, especialmente quando o SEO não é uma prioridade.

[Next.js](https://nextjs.org/) – Excelente para SEO e aplicações server-side. Para projetos menores, também permite integrar um backend de forma prática.

[Go](https://go.dev) – Linguagem de programação limpa e de alto desempenho. Atualmente estou estudando Go para utilizá-lo no desenvolvimento das minhas APIs.

[Express](https://expressjs.com/) – Enquanto ainda estou consolidando meus conhecimentos em Go, ou quando preciso de uma solução rápida e simples, utilizo Node.js com Express.

[PostgreSQL](https://www.postgresql.org/) – Banco de dados relacional que utilizo para armazenar dados em meus projetos. É robusto e confiável.

[SQLite](https://www.sqlite.org/) – Para projetos menores, como as visualizações dos posts no blog. É leve e fácil de usar.

[Tailwind](https://tailwindcss.com/) – Framework CSS que prioriza a praticidade. É fácil de aprender e torna a estilização muito mais rápida.

[shadcn/ui](https://ui.shadcn.com/) – Sempre que quero agilizar a criação de componentes sem precisar construir tudo do zero, essa é a biblioteca que escolho.

### Ferramentas

[Insomnia](https://insomnia.rest/) - Para testar APIs. 

[Filezilla](https://filezilla-project.org/) - Para transferir arquivos via FTP.

[Postico 2](https://eggerapps.at/postico2/) - Para gerenciar bancos de dados PostgreSQL.

[Notion](https://www.notion.so/) - Para anotações e organização de tarefas.

[Capcut](https://www.capcut.com/) - Para edição de vídeos. 

[OBS Studio](https://obsproject.com/) - Para gravação de tela.

### Serviços

[Vercel](https://vercel.app) - Utilizo para hospedar meus projetos em Next.js. É, basicamente, a forma mais fácil e prática de fazer deploy.

[HostGator](https://www.hostgator.com.br/) - Uso para hospedar minhas páginas SPA desenvolvidas com Vite.

[Hostinger](https://www.hostinger.com.br/) - Atualmente utilizo a Hostinger para hospedar minhas APIs. Como ainda não tenho projetos muito pesados ou com alto volume de requisições, ela atende bem para testes e pequenos projetos.
